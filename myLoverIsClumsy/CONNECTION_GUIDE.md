# Expo Go 연결 가이드

## 문제: "No apps connected" 또는 새로고침 버튼만 보임

### 1단계: Expo Go 앱 확인

#### Android 디바이스에서:
1. Play Store에서 **"Expo Go"** 앱 설치 확인
2. Expo Go 앱이 최신 버전인지 확인
3. Expo Go 앱 열기

#### iOS 디바이스에서:
1. App Store에서 **"Expo Go"** 앱 설치 확인
2. Expo Go 앱이 최신 버전인지 확인
3. Expo Go 앱 열기

### 2단계: 네트워크 연결 확인

**중요:** 컴퓨터와 모바일 디바이스가 **같은 WiFi 네트워크**에 연결되어 있어야 합니다!

1. 컴퓨터의 WiFi 이름 확인
2. 모바일 디바이스의 WiFi 이름 확인
3. **같은 네트워크인지 확인**

### 3단계: 연결 방법

#### 방법 1: QR 코드 스캔 (가장 쉬움)
1. 터미널에서 `npm start` 실행
2. QR 코드가 터미널에 표시됨
3. Expo Go 앱에서 "Scan QR code" 선택
4. QR 코드 스캔

#### 방법 2: URL 직접 입력
1. 터미널에 표시된 URL 확인 (예: `exp://192.168.0.3:8081`)
2. Expo Go 앱에서 "Enter URL manually" 선택
3. URL 입력

#### 방법 3: Tunnel 사용 (같은 WiFi가 아닌 경우)
```bash
# 터미널에서
npx expo start --tunnel
```
- 이 방법은 인터넷 연결이 필요합니다
- 조금 느릴 수 있습니다

### 4단계: 연결 확인

연결이 성공하면:
- 터미널에 "Connected to T811" 같은 메시지 표시
- 모바일에서 앱이 로드됨
- "No apps connected" 경고가 사라짐

### 5단계: 문제 해결

#### 여전히 연결되지 않는 경우:

1. **방화벽 확인**
   - Windows 방화벽이 포트 8081을 차단하지 않는지 확인
   - 방화벽 예외 추가

2. **포트 확인**
   ```bash
   # 다른 포트 사용
   npx expo start --port 8082
   ```

3. **Expo Go 앱 재시작**
   - Expo Go 앱 완전 종료
   - 다시 열기

4. **서버 재시작**
   ```bash
   # 서버 중지 (Ctrl+C)
   # 캐시 클리어하며 재시작
   npx expo start -c
   ```

5. **네트워크 재연결**
   - WiFi 끄기 → 켜기
   - 컴퓨터와 모바일 모두

### 6단계: 환경 변수 확인

연결은 되지만 "Something went wrong"이 나오는 경우:

1. `.env` 파일 확인
2. 환경 변수가 올바르게 설정되었는지 확인
3. 서버 재시작

### 빠른 체크리스트

- [ ] Expo Go 앱이 설치되어 있음
- [ ] 컴퓨터와 모바일이 같은 WiFi에 연결됨
- [ ] 터미널에서 `npm start` 실행 중
- [ ] QR 코드를 스캔했거나 URL을 입력함
- [ ] 터미널에 "Connected" 메시지 표시
- [ ] `.env` 파일이 올바르게 설정됨

### 추가 도움말

여전히 문제가 있으면:
1. 터미널의 **전체 로그** 확인
2. Expo Go 앱의 **에러 메시지** 확인
3. 네트워크 상태 확인
