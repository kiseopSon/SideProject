# iptime A604SR 포트 포워딩 설정 가이드

## 1. 라우터 관리 페이지 접속
1. 브라우저에서 `http://192.168.0.1` 접속
2. 관리자 계정으로 로그인 (기본: admin/admin 또는 설정한 비밀번호)

## 2. 포트 포워딩 설정 경로
**방법 1 (권장):**
- 고급설정 > NAT/라우터 관리 > 포트포워드 설정

**방법 2:**
- 고급설정 > 방화벽 > 포트포워드 설정

## 3. 포트 포워딩 규칙 추가

### 설정 항목:
- **규칙명/설명**: Spring Boot Gateway (아무거나 가능)
- **외부 포트 시작**: `8080`
- **외부 포트 끝**: `8080` (또는 범위로 설정)
- **내부 IP 주소**: `192.168.0.3` ⚠️ **정확히 입력!**
- **내부 포트 시작**: `8080`
- **내부 포트 끝**: `8080`
- **프로토콜**: `TCP` 또는 `Both`
- **상태**: `활성화` / `사용` 체크

### 중요 체크리스트:
✅ 내부 IP가 정확히 `192.168.0.3`인지 확인 (오타 주의!)
✅ 외부 포트와 내부 포트가 모두 `8080`인지 확인
✅ 프로토콜이 `TCP` 또는 `Both`로 설정되어 있는지 확인
✅ 상태가 `활성화` 또는 `사용`으로 체크되어 있는지 확인
✅ 설정 후 **적용** 또는 **저장** 버튼 클릭

## 4. 추가 확인 사항

### UPnP 설정 확인
- 고급설정 > NAT/라우터 관리 > UPnP 설정
- UPnP가 활성화되어 있으면 자동으로 포트가 열릴 수 있음

### 방화벽 설정 확인
- 고급설정 > 방화벽 > 방화벽 설정
- 방화벽이 너무 엄격하게 설정되어 있으면 포트가 차단될 수 있음

### DMZ 설정 (임시 테스트용)
- 고급설정 > NAT/라우터 관리 > DMZ 설정
- DMZ 호스트 IP를 `192.168.0.3`으로 설정하면 모든 포트가 열림
- ⚠️ 보안상 위험하므로 테스트 후 반드시 해제!

## 5. 설정 후 확인
1. 라우터 재부팅 (선택사항, 권장)
2. 포트포워드 설정 목록에서 규칙이 활성화되어 있는지 확인
3. 외부에서 접속 테스트: `http://180.233.237.67:8080`

## 6. 문제 해결

### 여전히 안 되면:
1. **라우터 재부팅**: 설정이 제대로 적용되지 않을 수 있음
2. **다른 포트로 테스트**: 8080이 ISP에서 차단될 수 있음 (8081, 9000 등)
3. **DMZ로 테스트**: DMZ 설정으로 접속이 되면 포트 포워딩 설정 문제
4. **같은 네트워크 내부 테스트**: `http://192.168.0.3:8080` 접속 확인
5. **포트 스캔 도구 사용**: 외부에서 포트가 열려있는지 확인
