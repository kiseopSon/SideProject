# iptime A604SR 포트 포워딩 설정 가이드

## 📋 단계별 설정 방법

### 1단계: 라우터 관리 페이지 접속
1. 브라우저에서 `http://192.168.0.1` 또는 `http://192.168.0.1:8080` 접속
2. 관리자 계정으로 로그인
   - 기본 계정: `admin` / `admin`
   - 또는 설정한 비밀번호

### 2단계: 포트 포워딩 설정 메뉴 찾기
**경로 1 (권장):**
```
고급설정 > NAT/라우터 관리 > 포트포워드 설정
```

**경로 2:**
```
고급설정 > 방화벽 > 포트포워드 설정
```

### 3단계: 포트 포워딩 규칙 추가

#### 설정 화면에서 "추가" 또는 "신규 등록" 클릭

#### 입력 항목:
| 항목 | 값 | 중요도 |
|------|-----|--------|
| **규칙명/설명** | Spring Boot Gateway | 낮음 |
| **외부 포트 시작** | `8101` | ⭐⭐⭐ |
| **외부 포트 끝** | `8101` | ⭐⭐⭐ |
| **내부 IP 주소** | `192.168.0.3` | ⭐⭐⭐⭐⭐ |
| **내부 포트 시작** | `8101` | ⭐⭐⭐ |
| **내부 포트 끝** | `8101` | ⭐⭐⭐ |
| **프로토콜** | `TCP` 또는 `Both` | ⭐⭐⭐ |
| **상태** | `활성화` / `사용` | ⭐⭐⭐⭐ |

#### ⚠️ 주의사항:
- **내부 IP 주소**가 정확히 `192.168.0.3`인지 반드시 확인!
- 오타가 있으면 작동하지 않습니다
- 외부 포트와 내부 포트가 모두 `8101`인지 확인
- 프로토콜은 `TCP` 또는 `Both` 선택
- **반드시 "적용" 또는 "저장" 버튼 클릭!**

### 4단계: 설정 확인
1. 포트포워드 설정 목록에서 방금 추가한 규칙 확인
2. 규칙이 **활성화** 상태인지 확인
3. 내부 IP가 `192.168.0.3`으로 정확히 표시되는지 확인

### 5단계: 라우터 재부팅 (선택사항, 권장)
- 설정이 제대로 적용되지 않을 수 있으므로 라우터 재부팅 권장
- 관리 페이지에서 "시스템 관리 > 재부팅" 메뉴 사용

## 🧪 테스트 방법

### 1. 로컬 네트워크 테스트 (같은 네트워크)
```
http://192.168.0.3:8101
```
→ 이게 안 되면: Windows 방화벽 문제

### 2. 외부 네트워크 테스트 (다른 네트워크)
```
http://180.233.237.67:8101
```
→ 이게 안 되면: 라우터 포트 포워딩 또는 ISP 차단

## 🆘 여전히 안 되면

### 방법 1: DMZ 설정으로 테스트 (임시)
1. 고급설정 > NAT/라우터 관리 > DMZ 설정
2. DMZ 호스트 IP를 `192.168.0.3`으로 설정
3. 적용 후 외부 접속 테스트
4. ⚠️ **테스트 후 반드시 DMZ 해제!** (보안상 위험)

### 방법 2: 다른 포트로 테스트
- ISP가 8101 포트를 차단할 수 있음
- 8081, 9000, 3000 등 다른 포트로 테스트

### 방법 3: UPnP 확인
- 고급설정 > NAT/라우터 관리 > UPnP 설정
- UPnP가 활성화되어 있으면 자동으로 포트가 열릴 수 있음
